Siemens Engineer 30 Interview Summary

Memory Application
发布日期:   2017-11-14

前言

上周去西安面试了西门子的工程培训生项目，10.30飞西安好在机票酒店都可以报销。

面试当天10.31，宾馆离公司很近，所以时间也很充分，提前了半小时8:30到的公司，也只有两个人到了。当了之后先是给一个胸牌以确定身份，到九点钟的时候大概有24个人总共，HR就招待我们进去，直接领进会议室里面。九点半会议正式开始，先是HR做了开场，也就是欢迎大家到来的一些客套话以及讲了讲西门子的历史，然后就是请SFAE王总讲话，正式启动面试。

##无领导小组讨论

面试总共有三个部分，无领导小组讨论、笔试、专业单面。无领导小组讨论的形式是分成5个小组，小组讨论30min，最后做5min的总结，所有应聘者面对同一个问题。9:55开始，面试的题目如下：

原来有一批产品文档需要提交给客户，但客户临时改了需求导致延期提交。客户指定的截止时间是这个周五晚上提交，而你所在的部门现在只有两个员工正常上班，有另外三个员工由于忙于其他项目不在公司，而按照正常进度需要五个人才能完成。

  1. 请问面对这样的问题，你有什么解决方法？
  2. 你的方法中的关键因素是什么？

就是这样一个问题，拿到题目是9:55，可以在5min内问主考官问题，10:00正式讨论，10:30开始汇报。我们组是5个人，刚开始也有人问了问题，诸如可不可以找其他部分的人来帮忙？，短期内培训人员有可能吗？，可不可以让客户延期提交时间？等等，为了给面试官留个印象，我也问了请问是专业性极强的工作内容吗？有没有部分工作比如说让文职人员代劳的？Pre的方式是：发了5张A1纸，可以把方案记录下来。

转眼间就开始正式讨论了，由于汇报纸的方向是靠近桌子一边的，那边那位同学就乘势直接拿起了纸笔做起了领导，他的特点是头很油，就叫他油头吧。油头先发制人想做team leader，我发现其他几个人都没有戴手表，就做了time keeper，先说了那我们先来2min梳理自己的想法，然后5min大家每人1min做个汇报，花15min讨论成果，5min让reporter演练一下，剩下一点时间就充当裕量吧。这些话自然都是在面经上面看到的，就直接用了起来。实际上大家也基本上是按这个步骤进行的，大家想到的解决方法也差不多：

  1. 两位必然加班；如果只有两个人，现在的工作量是原来的2.5倍。
  2. 让3个出差的工程师远程协助，用石墨文档或者钉钉之类的写作软件；
  3. 请其他部门人帮忙做一些不太核心的工作；
  4. 请客户适当延后ddl（备用）。

其他四个人的想法其实也差不多，但是油头提出了预算相关方面，其他几个人都没有想到。整个讨论过程，油头都想带领这个团队，但不得不说，他对于团队的引领不是很好，我觉得他并没有把关键因素highlight出来，相反他认为每个方案中的小项就是具体方案可以作为重点，就这一点我很不能赞同，向他提出了三次标记关键词，但他并没有去做。在整个讨论之中，我做time keeper自认为是很合格的，每个5min汇报点提醒进度，有的地方停滞不前就会站出来说我们的时间不多了，不如先放一放进行下一项。我发现捎带提了我之前提出的2.5倍工作量问题，那我就直接说了工作量的评估应该是解决整个问题的关键，分配任务以及人员调配都应该以及工作量的多少，这一点重申在旁边一直看的面试官应该注意到了。

不知道油头是什么背景，他的思维和我们另外几个不太一样，甚至在有的问题上出现了分歧，但在这个时候大家只能求同存异。他的话也很多，有用无用对半开吧，但记录工作都是他做的。有的时候我不得不把他拽回来，我还甚至提出了举手表决的方法确定两个方案的优先级，但是效果并不好，主要是大家并不愿意举手。

最终我们得到如下的结果：

  • 确定工作量，2.5倍工作量是整个人员调度和时间分配的根本；
  • 方案介绍，总共五个方案，前三个从自身出发，主要是加班问题；后两个从客户出发，延长ddl；
    1. 加班
    2. 出差人员在下班时间帮忙
    3. 其他部门人员做一些非核心工作
    4. 问客户会不会有其他需求，以合理延长时间
    5. 压缩工作量
  • 适用场景总结

方案得出距临报告还有5min，有人问了一句谁来汇报呢？大概过了0.5秒没有人回答，我就主动站起来说那我来吧我想大家都还是很想汇报的应该都是谦虚了一下就被我给抢到了。我大概演练了一下，花了三分钟厘清了一下思路。然后HR宣布Pre正式开始。我们组有人提议做第一个汇报的。很巧，没有人和我们抢。那自然我是第一个汇报的人了。

上去布置好展示用的纸就开始了，自己也没有紧张，就是之前讨论的方案，主要是想法思路以及整个方案的结构我觉得都讲的很清楚，自己也比较享受。但是！忽然小组成员在下面给我提醒时间，我才意识到Pre时间是5min，然而此时已经过了50%的时间，而只讲了30%的内容。不得不加快语速，紧张之下后面有一部分也乱了一点。最后时间应该是超了1min左右，但是没有人care，也没有人叫停。

接下来到考官问问题的时候了，比方说要是其他部门也在忙呢？，加班问题上有没有考虑过项目的预算？，那边有所工作量是什么意思？等等这样的问题，所有的问题都是我最先回答的，我的一句主要都是写在纸上面的，回答的内容也蛮少的。后来都有小组成员起来补充，补充的都还挺不错的。尤其是预算问题上，油头还站到讲台上面补充了，讲的很在理。就感觉自己的回答糟糟的。

下去之后才知道，时间提醒是面试官让组员示意的，所有组员也并没有意识到时间问题。真的是摔。总体感觉自己除了时间把握上还是正常发挥了。

第二组的Pre是正常最好的，并且做了一个思维导图这是我们所没有意识到的。演讲的思路也很清晰，每个想法下面也都有对应的方案，看到第二组之后就感觉自己GG了。不过好在接下来的三组不知道为何出乎意料的差，所有的想法都和我们差不多，就没有什么出彩的地方，自然没有人关注的到，这大概算是第一个发言的优势吧。所有Pre看完之后，心态稍稳了一点。如果不是时间问题，我应该还是很不错的。群面结束是11:45。

笔试+单面

群面之后是午餐时间，公司提供了汉堡和牛奶，然后下午一点开始笔试和专业面试。午休时间和同来的人聊了聊，发现基本上都是说是，西安本地上大学的居多，从上海来的包括我只有3个人，另外两个都是硕士，但他们名校Buff都不及我。总归我交的学生还是应该自信！

一点钟开始笔试，分强电弱电和软件类两种题目，因为我本身不是电气工程科班，就拿了软件类的题目，并且之前有做过其他公司的笔试。拿到题目之后，发现竟然简单的离谱，任何一个CS科班的都能拿满分，可惜我不能因为我不是。专业面试与笔试同时进行，我是第一个被叫去面试的。

三个面试官面一个candidate，面我的主考官超级nice。问题其实都是之前准备过的问题，其中有一个用英文回答你的强项和缺点个是什么？我用英文回答了大概有三分钟，最后有个例子不知怎么讲好一些，就完成回答之后用中文补充了，面试官也没有异议。还问了很多有关做过的项目，都是做过的东西讲起来还是很容易的。他们的问题问完之后，我追加了两个问题，表达了自己的热忱和决心，总共面试大概有35min。

总结

整个面试来说，除了汇报时间把控上，其他各个地方都算是正常发挥了，至于能不能拿到offer主要看通过率了，要的人少大概就被刷了，还是蛮可惜的。不过我觉得群面的通过率我们组肯定是最高的。

总体来说，西家对于人才的培养真的是很不错的，其实从这次面试就看得出来他们的态度。有的公司大概就是浪费一下午，面试十分钟，有面过一个也算是业内大佬了，自我介绍时面试官在看简历，问问题的时候也不问做过的项目随便了一点，只好最后问了两个明知故问的问题拖了几分钟时间，缓解尴尬。
